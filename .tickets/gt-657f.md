# Design carefully gated agent hooks and workflow events

Explore agent hooks and workflow events for go-ticket, such as status-change events, pre-close checks, definition-of-done reminders, git workflow prompts, and integration points for editor/agent tooling. Treat shell commands and repository-controlled hook execution as deferred until explicitly security-reviewed.

## Acceptance Criteria

Design separates passive event metadata, explicit user-invoked checks, user-level hooks, and repo-controlled hooks; repository-controlled process execution remains disabled unless a security review approves a narrow model; any proposed hook schema includes opt-in, dry-run/reporting, timeout, environment, PATH, argument, and audit/log behavior; docs compare safe alternatives such as event JSON output or explicit check commands; no implementation ticket for executable hooks is opened without pre-implementation security approval; post-design security review records what remains deferred.
