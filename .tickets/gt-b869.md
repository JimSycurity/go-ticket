# Report cross-branch and worktree ticket accuracy

Add read-first git/worktree reporting that helps humans and agents detect ticket drift across branches, unpushed commits, dirty worktrees, stale status, conflicting ticket edits, and branch-local ticket state before acting on the wrong context.

## Acceptance Criteria

A command or documented workflow reports current branch, upstream/ahead/behind state, dirty ticket files, untracked ticket files, and where feasible conflicting ticket IDs across local branches/worktrees; output is safe for agents and does not mutate git state; docs explain limitations and when to run before implementing or closing tickets; tests use temporary git repositories for clean, dirty, ahead, and divergent-ticket cases; post-implementation security review confirms git integration does not execute hooks or mutate branches unexpectedly.
