# Add native query filters and stable output contracts

Expand gtk query beyond all-ticket JSONL by adding a reviewed native filter strategy and stable output contracts for agents, scripts, vscode-tk, and future TUI work. Avoid external jq or shell execution unless a separate security review explicitly approves it.

## Acceptance Criteria

Query supports documented native filters for common fields such as status, type, priority, assignee, tag, parent, dependency, linked ticket, external reference, and text/title where appropriate; output shape remains stable, path-free by default, and documented; unsupported filter syntax fails with actionable guidance; tests cover positive filters, no-match filters, malformed tickets, output shape, and rejected unsupported syntax; compatibility docs are updated from partial query parity to the implemented status; post-implementation security review confirms no unsafe process execution or path disclosure was introduced.
