# go-ticket

Cross-platform Go reimplementation of [`wedow/ticket`](https://github.com/wedow/ticket).

`go-ticket` aims to preserve the git-native `.tickets/` Markdown/YAML workflow
while making the core CLI usable on Windows, macOS, and Linux without Bash,
coreutils, jq, grep, or ripgrep.

## MVP Scope

The MVP binary is named `gtk` so it can be tested side-by-side with upstream
`tk`.

Planned MVP commands:

- `gtk help`
- `gtk init`
- `gtk create`
- `gtk show`
- `gtk list` / `gtk ls`
- `gtk start`, `gtk close`, `gtk reopen`, `gtk status`
- `gtk dep`, `gtk undep`, `gtk link`, `gtk unlink`
- `gtk ready`, `gtk blocked`
- `gtk add-note`
- `gtk version` / `gtk --version`

The MVP intentionally omits plugin execution, `query`, `edit`, `migrate-beads`,
`closed`, `dep tree`, `dep cycle`, shell completions, release artifacts, and a
drop-in `tk` executable name. Those belong to the feature-parity and
release-readiness phases.

`gtk list --json` is intended for local tools and editor integrations. Its
absolute path fields are local-machine metadata and can expose usernames or
repository locations when copied into logs, tickets, or reports.

## Development

```sh
go test ./...
go build -o /tmp/gtk ./cmd/gtk
/tmp/gtk version
```

The test suite includes fixtures generated by upstream `tk` plus hand-authored
edge fixtures for portability and malformed-file behavior.

Go embeds VCS metadata in normal local builds when it can see the repository.
For ad hoc smoke binaries, `gtk version` reports the version label, VCS
revision, dirty state, executable path, and executable mtime so `/tmp/gtk` can
prove which file is being smoke tested. A temporary explicit label and build
timestamp can be added with:

```sh
go build -ldflags "-X github.com/JimSycurity/go-ticket/internal/app.Version=dev-security-review -X github.com/JimSycurity/go-ticket/internal/app.BuildDate=$(date -u +%Y-%m-%dT%H:%M:%SZ)" -o /tmp/gtk ./cmd/gtk
```
